Primeros pasos

El primer paso de este ejercicio-tuturial es crear el repositorio. git init

Si bien podríamos clonar un repo remoto, la idea es hacerlo de otra forma git remote add origin https://github.com/nico88/IDS-HGM En la sentencia anterior, podemos asignarle al repositorio remoto el nombre que creamos conventiente para nuestro trabajo. Sin embargo, para este ejercicio particular, nombrarlo ORIGIN tendrá algunos beneficios.

Obtener los archivos a modifircar

Obtener la rama master git pull origin master

Creamos una rama nueva git checkout -b nombre_rama

Divertite poniendo colores!

OK. Te gusto lo que hicite? mandalo al server! git add * git commit -m "un mensaje para el grupo" git push origin nombre_rama Con el comando anterior subiste tu rama al server (solo la rama nombre_rama).

Revisión de todas las ramas

Obtenemos todas las ramas remotas git fetch --all

Probamos las distintas ramas para ver las COLORIDAS propuestas git checkout nombre_de_la_rama_a_checkear

Encontramos la rama ganadora y decidimos hacer los cambios efectivos. git checkout master git merge nombre_rama_elegida --commit

Enviamos la nueva version al servidor git push origin master

NOS ARREPENTIMOS!!!

Decidimos que la versión original era "más mejor" y LA VAMOS A RECUPERAR! git reset --hard commit_sha

Enviamos la "nueva vieja" rama al servidor git push origin master UPS! git push --force origin master

Comandos útiles

  • git remote -v Muetra todos los repositorios
  • git branch -a Muestra todas las ramas locales y remotas
  • git fetch --all Obtiene todas las ramas
  • git log --oneline Muestra los commits en una sola linea
  • git log --graph Muestra los commits y los merge en forma de arbol
  • git log --stat Muestra las estadisticas de cambios
  • git merge rama_merge --commit Une la rama actual con rama_merge y realiza un commit
  • git reset --hard commit_sha
